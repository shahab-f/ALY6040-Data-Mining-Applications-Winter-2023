Therefore, in situations where imbalanced data is encountered, various techniques such as oversampling, undersampling, or synthetic sampling can be used to balance the data and improve the performance of the predictive models.

It is generally recommended to perform oversampling or any other data preprocessing steps after splitting the data into training and testing sets. This is because oversampling the minority class(es) before splitting the data can result in the same observations being present in both the training and testing sets.

Having the same observations in both the training and testing sets can lead to overfitting and unrealistic estimates of model performance. Therefore, it is important to ensure that the training and testing sets are independent and identically distributed.

In the code provided earlier, oversampling is applied to the train_data dataset after it has been split from the original data using createDataPartition. This helps ensure that the oversampled data is used only for training the model, and that the testing set is not influenced by the oversampling process.

svm_model <- train(

  method = "svmRadial",
