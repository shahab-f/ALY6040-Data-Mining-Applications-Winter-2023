number.perfect.splits <- apply(X=mushrooms[-1], MARGIN = 2, FUN = function(col){
  t <- table(mushrooms$class,col)
  sum(t == 0)
})

# Descending order of perfect splits
order <- order(number.perfect.splits,decreasing = TRUE)
number.perfect.splits <- number.perfect.splits[order]

CODE WALK THROUGH

The code above first creates a table showing the frequency of each combination of class and odor in the dataset. It then calculates the number of perfect splits each column in the dataset can achieve when used to separate the two classes of mushrooms based on their odor. Finally, it sorts the columns in descending order based on their number of perfect splits, allowing for identification of the most useful variables for separating the two classes of mushrooms based on their odor.

To explain, Perfect splits are like magic rules that can help us tell if a mushroom is poisonous or not, just by smelling it. We want to find the best rule that always works, so we look at all the different rules we can make by using different smells as clues. The code counts how many times we can make a perfect rule with each smell. If a smell has a lot of perfect rules, that means its a really good clue to tell if a mushroom is poisonous or not. So we use this code to find the smells that are the best at helping us tell the difference between good and bad mushrooms (CART - classification and regression trees, 2021).

# penalty matrix
penalty.matrix <- matrix(c(0,1,10,0), byrow=TRUE, nrow=2)

# building the classification tree with rpart
tree <- rpart(class~.,
              data=mushrooms_train,
              parms = list(loss = penalty.matrix),
              method = "class")

# tabulating the result
#table(tree$frame$var, tree$frame$yval)

CODE WALK THROUGH

The code above creates a classification tree model with a custom penalty matrix that assigns different penalties to different types of misclassifications (false negatives have a penalty of 1, false positives have a penalty of 10 and correct predictions have no penalty). The resulting model can be used to predict the class of mushrooms and assess the risk of making different types of misclassifications. The parms argument specifies additional parameters, with loss = penalty.matrix indicating use of the custom penalty matrix from previous step.

# Visualize the decision tree with rpart.plot
rpart.plot(tree, nn=TRUE)

OBSERVATION

The tree is a flowchart-like structure that represents the different decisions that the model makes to classify mushrooms as either poisonous or edible. At each decision point, the tree considers a specific feature of the mushroom (e.g., odor, spore-print-color, population, gill-size, habitat) and makes a decision based on the value of that feature. Here is how to interpret the decision tree in the context of a mushroom dataset (Introduction to R: Decision trees, n.d.):

  1. The first decision is based on the odor of the mushroom. If the odor is not a, l, or n, the mushroom is classified as poisonous, and the decision-making process stops. Otherwise, the model moves on to the next decision.

  2. The second decision is based on the spore print color of the mushroom. If the spore print color is b, h, k, n, o, u, or y the mushroom is classified as edible, and the decision-making process stops. Otherwise, the model moves on to the next decision.

  3. The third decision is based on the mushrooms population. If its not c, n, s, or y, the probability of it being poisonous is 0.64; otherwise, the probability is 0.03. The model then moves to the next decision in both cases.

  4. The fourth decision is based on the size of the mushrooms gills. If the gills are b, the mushroom is classified as edible, and the decision-making process stops. Otherwise, the mushroom is classified as poisonous.

  5. The fifth and final decision is based on the habitat of the mushroom. If the habitat is l or p, the mushroom is classified as edible. Otherwise, the mushroom is classified as poisonous and the decision-making process stops.

Overall, the decision tree provides a set of rules that can be used to predict whether a mushroom is poisonous or not based on its features. By following the decision path from the root node to the leaves, you can determine the final prediction for any given mushroom.

# choosing the best complexity parameter "cp" to prune the tree
cp.optim <- tree$cptable[which.min(tree$cptable[,"xerror"]),"CP"]
# tree prunning using the best complexity parameter. For more in
tree <- prune(tree, cp=cp.optim)

This code first identifies the best complexity parameter for pruning a decision tree using cross-validation. It then prunes the tree using this parameter to improve its predictive performance on new data.
